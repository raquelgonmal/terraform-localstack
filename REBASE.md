# Limpieza de commits con rebase

## Proceso realizado
Se utilizó el comando git rebase -i HEAD~3 para modificar el historial de commits.

## Cambios realizados
- Se editaron mensajes de commits para hacerlos más descriptivos.
- Se fusionaron commits innecesarios usando squash.

## Resultado
Se obtuvo un historial limpio con un único commit representativo.

## Aprendizaje
El uso de git rebase permite mantener un historial más claro y profesional.
